# Family Life Management System

A Claude skill-based system for managing the cognitive load of a busy family. See `REQUIREMENTS.md` for full design context.

## Getting Started

Run **`/family-init`** to set up your household. It walks you through a discovery conversation and generates all the personal config files the system needs. Everything personal lives in `data/` which is gitignored.

## Commands

- **`/family-init`** - One-time setup. Generates config from a household discovery conversation.
- **`/family-plan`** - Weekly planning session (Sunday/Monday). Conversational Q&A that produces a checklist and insights.
- **`/family-checkin`** - Evening check-in (~5 min). Structured Q&A to log what happened today.
- **`/family-status`** - Quick glance at current state across all domains.
- **`/family-log`** - Quick capture. No Q&A, just confirms what it captured.

## Key Principles

- Primary caregiver is analog (paper checklists). Tech partner is the interface.
- Talking points, not prescriptions. Surface observations, don't dictate.
- Checklist, not calendar. Organized by what needs doing, not when.
- All cadences and thresholds are tunable. This is iterative.

## Data Layout

```
data/                        # gitignored - all personal data stays local
├── config/                  # Generated by /family-init (edit to tune)
│   ├── domains.yaml         # Domain definitions, cadences, thresholds
│   ├── schedule.yaml        # Fixed weekly commitments
│   └── laundry-loads.yaml   # Load types and pipeline stages
├── state/                   # Running state (updated by check-ins)
│   └── current.yaml
└── weeks/                   # Weekly plans and daily check-in logs
    └── YYYY-WXX/
        ├── plan.md
        ├── insights.md
        └── checkins/
```

## Two Audiences

- **Caregiver's output**: Bullet points, checkboxes, no prose. Designed to be copied to paper.
- **Tech partner's output**: Conversational, insights, pattern observations, talking points.
